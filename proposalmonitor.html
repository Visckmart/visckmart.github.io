Proposal Monitor

Keep up to date with Swift's language proposals

Developer Tools

Proposal Monitor lets you explore all of the proposals made for the Swift programming language, see their status and details such as a summary and related previous proposals.

It also makes it easier than ever to keep up to date with new proposals and changes to the ones being discussed.

This project is entirely developed and maintained by myself, from the interface to the implementation of the app and the backend for the notifications and proposal activity history.

Features

Notifications

  • – Get notified about proposal updates with natural sentences and start reading about them on the spot!
  • – Fine tune the notifications you want to receive.

Widgets

  • – See the latest proposal update on your home screen throughout the day.
  • – Available in multiple sizes, showing details about the update and information about the authors and review managers on specific sizes.

Activity

  • - The Activity tab allows users to see the recent proposal activity chronologically.
  • - The cloud backend makes sure that even brand-new users are able to load the recent activity history.

Interface

  • - The interface is cohesive between all platforms (iOS, iPadOS and visionOS).
  • – At the same time, the interface respects each of their specific visual identities and takes advantage of platform-specific features such as ornaments on visionOS.

Search & Filters

  • – Filters let you quickly see what's being reviewed and what's coming on the next version.
  • – Search lets you find proposals by searching for their title, summary, author name, the version a proposal was implemented and much more.
  • - Proposals show up in Spotlight, allowing users to open them up with ease.
The app was developed with full support for Dynamic Type and VoiceOver in mind to ensure great Accessibility.

Metodologia

O aplicativo foi desenvolvido por participantes do programa Apple Developer Academy PUC-Rio. Por conta disso, utilizamos a metodologia CBL (Challenge Based Learning). Tivemos aproximadamente 3 meses entre o anúncio dos grupos, as discussões entre integrantes para a decisão do tema do aplicativo, o seu desenvolvimento, sua publicação e a apresentação final.

Backend

The backend is built on AWS using mainly AWS Lambda to fetch the API and check for updates, Amazon DynamoDB to help the change detection and AWS SNS to push silent push notifications to the devices. The AWS Lambdas are written in python, although I already made experiments with running it on Swift too.

The app's source of truth is the official Swift Evolution Metadata API. I participated actively in the process the Swift Website Workgroup (SWWG) initiated to shape this API.
